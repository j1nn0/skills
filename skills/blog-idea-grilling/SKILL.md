# blog-idea-grilling

## 企画の芯

**企画の芯**は、書き手の材料で支えられる「一人の読者が得る変化」と「そのための一文の主張」である。このスキルはアイデアを採点せず、企画の芯が見える形まで育てる。

出力は記事本文ではなく、次工程で使う企画カードにする。

## 育てる

1. `blog-ops` から渡されたリポジトリの制約と記事の段階を引き継ぐ。既存記事・メモ・ファイルから確認できる事実は調べ、企画を変える選択だけをユーザーに尋ねる。

   完了: 今ある素材、決まっていること、次に決める一点を区別できる。

2. 素材、読者の変化、主張、根拠と範囲のうち、企画の芯を最も前に進める一点を選ぶ。一度に一問だけ尋ねる。質問には、なぜその選択が必要かと、現在の材料からのおすすめを添える。

   ```markdown
   ### 問い: [決めること]

   [答えやすく具体化した一問]

   - 理由: [この答えが企画をどう変えるか]
   - おすすめ: [現在の材料に基づく仮説]
   ```

   完了: 読者、読後の変化、主張、支える材料、範囲を一貫して説明できる。

3. 答えによって元の方向が弱くなったら、利用できる材料を保った代替の切り口を示す。対象読者を絞る、主張を経験に寄せる、比較を実践記へ変える、といった方向から、書ける企画を選ぶ。

   完了: 材料から無理なく支えられる企画が一つ選ばれている。

4. 企画の芯がそろったら企画カードを作り、`blog-writing-guide-ja` へ渡す。確認が残る事実には `要確認` と確認方法を書く。

   完了: 次の工程が、構成づくりか素材の確認かを一つ選べる。

## 企画カード

```markdown
## 企画カード

- 仮題: [読者と価値が伝わる題]
- 想定読者: [具体的な状況]
- 読後の変化: [できるようになること]
- 中心の主張: [一文]
- 支える材料: [経験、検証、比較、一次情報]
- 扱う範囲: [本文で扱うこと]
- 境界と要確認事項: [扱わないこと、未確認の事実と確認方法]

### 構成の芯

1. [節の役割]
2. [節の役割]
3. [節の役割]

### 次の一手

[blog-writing-guide-ja で構成を作る、または確認すべき素材を集める]
```
